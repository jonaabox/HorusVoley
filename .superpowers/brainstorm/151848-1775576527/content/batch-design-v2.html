Feature #1 — Batch notificación deudores (v2)

Con días de aviso configurables desde Configuración

Cambio en el modelo de datos

Se agrega una nueva clave en la tabla configuracion:

dias_aviso_vencimiento = '5' (valor por defecto)

La sección Configuración ya muestra campos editables por clave. Se agrega este campo junto a "Día de vencimiento de cuota".

¿Cómo funciona el "batch"?

Al cargar la app, DebtAlertContext consulta Supabase para obtener alumnos activos, sus pagos y la config. Calcula qué alumnos no pagaron el mes actual y tienen el vencimiento a ≤ N días (configurable). El resultado queda en contexto global, disponible en toda la app sin nuevas consultas.

Flujo completo

App carga
→
Consulta alumnos + pagos + config
→
Calcula deudores próximos
→
Badge + banner en Sidebar
→
Click → Campañas pre-cargada

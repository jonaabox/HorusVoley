Feature #4 — Enfoques de implementación

3 formas de reemplazar los confirm() y alert() en el proyecto

A

Un componente ConfirmModal reutilizable

Se crea un único componente que acepta props: title, message, onConfirm, onCancel, variant (danger/info). Cada página lo monta en su propio estado local. 3 archivos modificados + 1 componente nuevo.

Pros

  • Simple de entender
  • Sin contexto global
  • Cada página es independiente

Cons

  • Estado repetido en cada página
B ⭐ Recomendado

Hook useConfirm + componente

Un hook que devuelve { confirm, ConfirmModal }. En cada página: const { confirm, ConfirmModal } = useConfirm() y se llama await confirm({...}) como si fuera el confirm() nativo. El componente se monta al final del JSX.

Pros

  • API casi idéntica al confirm() original
  • Limpio: no poluciona el estado del componente padre
  • Fácil de usar en el futuro

Cons

  • Ligeramente más complejo que A
C

Context global (DialogProvider)

Un provider global en App.jsx que expone useDialog(). Un solo modal en toda la app, controlado desde cualquier componente sin montar nada localmente.

Pros

  • Un solo modal en todo el DOM
  • Se puede llamar desde cualquier lado

Cons

  • Más arquitectura para un cambio pequeño
  • Más difícil de debuggear
